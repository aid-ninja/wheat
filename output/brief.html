Multi-Session

13 claims | tested evidence | 4 sources | prototype working

Prototype Validated

SessionState class, session-keyed Maps, getSession(id) lazy init, SSE tagged with session_id, dashboard session pills with hue-from-hash. Backwards compatible with single-session via 'default' fallback p003.

Architecture

Single server, Map<session_id, SessionState> with lazy initializer r003 r004. Hook payloads include stable UUID session_id r001. SessionStart/SessionEnd hooks provide explicit boundaries r002 w001. Heartbeat timeout (5min) for crashed sessions r007.

Constraint: Lifecycle hooks are command-only, not HTTP w002 — curl wrapper required.

Dashboard UX

Unified feed with session color-coding (hue from hash). Pill-bar session switcher. No split views on mobile r005 p002. Estimated 2–4 hours effort r008.

Risk: Approval Confusion

Two sessions needing approval simultaneously — user could approve wrong one. Mitigated with prominent session labels + colors r006.

Security

12 claims | tested evidence | 5 sources | hardening implemented

Mitigations Implemented

Cookie-based auth (HttpOnly, SameSite=Strict) with 302-redirect to clean URL. CORS restricted to request origin. Referrer-Policy: no-referrer. X-Content-Type-Options: nosniff p005.

SSE Broken Through Quick Tunnels

CF quick tunnels do not support SSE — dashboard falls back to 2s polling r027. Quick tunnels are dev-only with 200 req limit r028.

Token Compromise Blast Radius

If token leaks: approve arbitrary tool executions, set autonomous mode, inject /api/ask responses, read all data r031. No token rotation — restart is only fix r034.

Remaining Gaps

  • CSRF tokens for state-changing endpoints r035
  • Named tunnel or WebSocket transport r036
  • Token rotation mechanism r034
  • Rate limiting on auth failures r033

State Persistence

5 claims | documented evidence | 2 sources | new topic from blind-spot analysis

All State In-Memory

Server restart = total loss of trust levels, session rules, activity log, and all session state. No SIGTERM/SIGINT handlers r038.

Recommended: JSON Snapshots

Persist trust levels, rules, and activity log to farmer-state.json. ~40 LOC using built-in fs module (zero deps) r039. Pending permissions are ephemeral — do NOT persist r040.

Graceful Shutdown

SIGTERM/SIGINT handlers: flush state, deny pending permissions with 'server shutting down', broadcast shutdown SSE event r041. Event sourcing is over-engineered at this scale r042.
