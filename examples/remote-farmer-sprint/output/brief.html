How It Works

Claude Code CLI
  |
  |-- PreToolUse hook ----------POST--> localhost:9090/hooks/permission
  |-- PostToolUse hook ---------POST--> localhost:9090/hooks/activity
  |-- PostToolUseFailure -------POST--> localhost:9090/hooks/activity
  '-- Notification hook --------POST--> localhost:9090/hooks/notification
                                              |
                                    Remote Farmer Server (Node.js)
                                              |
                                    SSE push + polling fallback
                                              |
                                       Browser Dashboard
                                    (approve / deny / monitor)

Server holds the HTTP response open. Browser shows Allow/Deny. Decision relays back synchronously. Claude Code proceeds or halts. [r001] [p002]

Cloudflare buffers SSE — dashboard auto-detects and falls back to polling every 2s. State diff prevents UI flash. [p007] [p011]

Trust Tiers

Paranoid
Approve everything
Every tool call requires manual approval. Maximum control for security-sensitive operations.
Standard
Auto-approve reads
Read, Grep, Glob, WebSearch, WebFetch pass through. Writes and execution need approval.
Autonomous
Auto-approve all (safe)
Everything auto-approved except dangerous Bash: rm -rf, git push, sudo, curl|sh. [p009]

Session rules via checkbox UI. Quick Rules dropdown on each permission card. Eliminates approval fatigue from ~120s/call to zero for safe tools. [r028] [r031]

Security Model

Critical Fix
Fail-deny on timeout
Empty JSON {} on 2xx = auto-allow in Claude Code. Timeout now returns explicit deny. [x001] [x008]
Known Limitation
Fail-open on crash
Server crash = permissions silently granted. Unfixable from dashboard side. Mitigate with local command hook. [x002] [x010]

Hardening

Hook endpoints: localhost only [p010]

Body size: 1MB cap

Token: timing-safe comparison

Patterns: regex-escaped (no ReDoS)

Dangerous commands: red visual flag

Auth Layers

Inner: Hex token on all API/SSE access

Outer: Cloudflare Zero Trust email OTP

Tested Findings

Cloudflare Tunnel

SSE buffered by proxy
Despite X-Accel-Buffering:no headers, Cloudflare buffers SSE data. Dashboard auto-falls back to polling. [p007]
End-to-end verified
Quick tunnel to iPhone, remote approve/deny from phone, live activity feed — all working. [p012]

iOS Safari

No Notification API
typeof Notification === 'undefined'. All references must be guarded. [p008]
Dynamic Island safe area
env(safe-area-inset-top) + viewport-fit=cover required for proper header positioning.

Limitations

Blocked
No remote text input
Hooks can't answer AskUserQuestion prompts. Text conversation stays in terminal. [r023]
Unfixable
Fail-open on crash
Server down = Claude Code proceeds freely. Command hook is best workaround. [x002]
Waiting
No hot-reload
Hook config loaded at startup. Changes require session restart. [r007]

Recommendation

Ship It
The 650-line prototype covers the core use case
Remote permission approval from a phone over a tunneled connection. Trust tiers solve approval fatigue. Polling fallback handles proxy compatibility. Zero npm dependencies.
