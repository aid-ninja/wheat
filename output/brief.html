Security

16 claims | tested evidence | 6 sources | hardening + WebSocket + CSRF + WS hardening

Mitigations Implemented

Cookie-based auth (HttpOnly, SameSite=Strict) with 302-redirect to clean URL. CORS restricted to request origin. Referrer-Policy: no-referrer. X-Content-Type-Options: nosniff p005.

WebSocket Transport + Hardening

RFC 6455 handshake + frame encoding/decoding (~120 LOC, zero deps). Cascade fallback: WebSocket first (CF tunnels), SSE second (local), 2s polling baseline p009.

Hardened p011: Origin validation x004, token as first WS message x005, 1MB buffer cap x006, FIN bit + continuation frames x007, RFC 6455 close opcode x008, exponential backoff reconnect x009. All 6 risks resolved.

CSRF Protection + Token Rotation

Per-session CSRF tokens (crypto.randomBytes, 24h TTL). All mutating dashboard endpoints validate X-CSRF-Token header. Configurable auto-rotation + on-demand rotation via POST /api/admin/rotate-token. Retired tokens have grace period with timing-safe comparison p010.
