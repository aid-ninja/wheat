Why Scale?

The Problem

Remote Farmer works for one session, one sprint. That is not enough.

Context Pollution
157 files
Repo already at 157 files (not 80 as estimated). Grep for "claims.json" returns 71 matches. The examples/ dir creates 62 duplicate-named files. x001 x002
Single-Session Lock
One session at a time
Global singletons (pending, activity, trustLevel) mean only one Claude Code session can use the dashboard. Two sessions = state corruption. r004
No Sprint Switching
Flat file structure
Archived sprints live in examples/ with no dashboard access. No way to view, compare, or reference across sprints. d002
The hidden cost: Context window pollution -- not I/O latency -- is the real bottleneck. Noisy tool results waste 10-40K tokens per research session, 5-20% of the context budget. x003

Risk Register

Risks & Mitigations

Top 5 identified risks across all three workstreams.

1

Approval Confusion (Multi-Session)

Two sessions needing approval simultaneously -- user could approve the wrong one. Mitigate: prominent session labels + distinct colors + audio/haptic differentiation. r006

2

Symlink Portability (Multi-Sprint)

Git symlinks break on Windows without admin privileges. Mitigate: wheat.config.json is canonical pointer; symlinks are optional local convenience. r025

3

CLAUDE.md Staleness (Cartography)

Per-directory CLAUDE.md files claiming contents that have moved actively mislead AI. Mitigate: CLAUDE.md for behavior only; manifest (auto-generated) for structure. r015

4

Context Window Pollution (Performance)

Noisy Glob/Grep results waste 10-40K tokens per session (5-20% of context budget). Mitigate: manifest eliminates iterative search; already past crossover point. x003

5

Lifecycle Hook Limitation (Multi-Session)

SessionStart/SessionEnd are command-only -- cannot POST directly to Farmer. Mitigate: curl wrapper shell script; trivial but must be documented. w002
