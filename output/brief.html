Scaling Remote Farmer

Decision Brief | 2026-03-12 | 35 claims | 5 topics

Three independent workstreams to scale Remote Farmer: multi-session via session-keyed Maps, multi-sprint via sprints/ directory with config pointer, and repo cartography via compiler-generated topic-map manifest.

The manifest generator is already built and tested at <10ms p001. Multi-session refactor is planned at ~300 lines p002. Total effort: 5-9 hours r008 r026.

Multi-Session

12 claims | documented evidence | 4 sources

Architecture

Single server, session-keyed state: Map<session_id, SessionState> with lazy getSession(id) initializer r003 r004. Hook payloads already include stable session_id UUID r001.

Lifecycle

SessionStart/SessionEnd hooks provide explicit boundaries r002 w001. Heartbeat timeout (5min) for crashed sessions r007.

Constraint: Lifecycle hooks are command-only, not HTTP w002 — curl wrapper required.

Dashboard UX

Unified feed with session color-coding (hue from hash). Pill-bar session switcher. No split views on mobile r005 p002.

Risk: Approval Confusion

Two sessions needing approval simultaneously — user could approve wrong one. Mitigate with prominent session labels + colors r006.

Multi-Sprint

8 claims | documented evidence | 2 sources

Data Model

Each sprint in sprints/<slug>/, self-contained with own claims.json r020. Current sprint pointer in wheat.config.json (not symlinks) for portability r025.

Dashboard

Collapsible sprint card list — shows question, phase badge, claim count r022. Server adds --sprints-dir with hot-reload r021.

Cross-Sprint References

Format: sprint-slug:claim-id (e.g., remote-farmer:r003). No schema changes needed r024.

Resolved: Symlinks vs Config

r020 proposed symlinks. r025 flagged Windows portability. Resolution: config file is canonical, symlinks are optional convenience.

Repo Cartography

8 claims | tested evidence | 4 sources

Manifest Generator (BUILT)

wheat-manifest.json — topic map generated in <10ms, ~5KB p001. Maps topics to claims, files, and sprints. Single Read replaces 3-5 Glob/Grep calls r012.

Design

Topic map, not file tree — AI searches by concept r017. Auto-generated by compiler so it never drifts r011. CLAUDE.md stays behavioral, manifest is structural r015.

Urgency: Cartography Needed NOW

Repo already at 157 files (not 80 as estimated) x001. The examples/ dir creates 62 duplicate-named files causing search noise x002. Context window pollution — not I/O — is the real cost x003.
