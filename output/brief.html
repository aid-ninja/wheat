Security

14 claims | tested evidence | 5 sources | hardening + WebSocket + CSRF implemented

Mitigations Implemented

Cookie-based auth (HttpOnly, SameSite=Strict) with 302-redirect to clean URL. CORS restricted to request origin. Referrer-Policy: no-referrer. X-Content-Type-Options: nosniff p005.

WebSocket Transport

RFC 6455 handshake + frame encoding/decoding (~120 LOC, zero deps, Node 20 compatible). Dashboard cascade fallback: WebSocket first (works through CF quick tunnels), SSE second (works local/direct), 2s polling baseline p009. 30s ping/pong heartbeat. Auth via timing-safe token comparison. SSE preserved for backwards compat.

CSRF Protection + Token Rotation (NEW)

Per-session CSRF tokens (crypto.randomBytes, 24h TTL) delivered via init payloads and /api/state polling. All mutating dashboard endpoints validate X-CSRF-Token header, returning 403 on mismatch. Hook endpoints (localhost-only) exempt p010.

Configurable auto-rotation (--token-rotation-interval) and on-demand rotation via POST /api/admin/rotate-token. Retired tokens have configurable grace period (--token-grace-period, default 60s) with timing-safe comparison. Dashboard auto-updates cookie on rotation broadcast. Zero deps, ~80 LOC.

Token Compromise Blast Radius

If token leaks: approve arbitrary tool executions, set autonomous mode, inject /api/ask responses, read all data r031. Now mitigated: token rotation + CSRF tokens reduce blast radius and enable recovery without restart p010.

  • Rate limiting on auth failures r033

State Persistence

6 claims | tested evidence | 3 sources | implemented and verified

JSON Persistence (DONE)

Atomic writes (tmp+rename) to .farmer-state.json. Restores trust levels, session rules, and activity logs on restart. 30s periodic auto-save. ~45 LOC, zero dependencies. Verified: state survives full stop/start cycle p008.

Graceful Shutdown

SIGTERM/SIGINT handlers: flush state, deny pending permissions with 'server shutting down', broadcast shutdown event r041 p008.

Design Decisions

State classified into must-persist vs ephemeral r040. Pending permissions, SSE connections correctly excluded. Event sourcing over-engineered at this scale r042. JSON snapshots are sufficient r039.
