Cognitive Load Theory

The Worked Examples Effect

John Sweller (1980s): studying solved examples reduces cognitive load more effectively than solving equivalent problems. Learners avoid weak strategies like means-ends analysis that overwhelm working memory.

Good news: The Wheat replay IS a worked example — a complete, solved research sprint.

Bad news: The split-attention effect means separated text and diagrams increase extraneous load. The replay shows transitions without inline explanation, forcing learners to mentally integrate what they see with what it means.

Implication
Annotations must be integrated directly into the visualization
Not in a separate panel, not in docs — inline, contextual, triggered by the frame being viewed. This is the split-attention fix.

Progressive Disclosure

Incremental Is Not Scaffolded

ApproachMeaningWheat?
IncrementalShows more data over timeYes — claims accumulate
ScaffoldedGates complexity by conceptual dependencyNo — everything shown at once per frame

Progressive disclosure (NN/g) improves learnability, efficiency, and error rate by synchronizing complexity with user involvement. A scaffolded replay ensures learners understand claim types before evidence upgrades, and evidence tiers before conflict resolution.

Key Distinction
Synchronize revealed complexity with the learner's current understanding
Beginners don't need to see compiler gates. But hiding them forever frustrates advanced users. The solution: conceptual acts with optional gating.

Explorable Explanations

Victor & Case: Four Principles

1. Compelling Question
"Traditional teaching fails because it answers questions the student hasn't thought to ask." — Nicky Case
2. Isolate Mechanics
Teach individual cause-and-effect relationships one at a time before combining them.
3. Let Readers Create Data
Active exploration beats passive observation. Let learners create their own claims and see the system respond.
4. End with Open Questions
The learner should be able to go beyond the teacher — exploring questions even the author didn't anticipate.

The Wheat replay should open with: "How does a research sprint turn messy exploration into a structured recommendation?"

Prior Art

Tools That Teach by Showing Themselves

LearnGitBranching
Visualize + Interact
100% client-side. Commit trees update in real-time. Dual mode: sandbox for exploration, structured levels for isolated mechanics. The visualization IS the learning.
Redux DevTools
Make Invisible Visible
Time-travel debugging, action inspection, state diffs. Developers learn state patterns by seeing and manipulating them. The tool teaches by making the implicit explicit.
Observable
Interleave Prose + Code
Reactive documents where explanation IS running code. Inline annotations explain each cell. Readers modify and see results immediately.

Shared pattern: Make the invisible visible, let users manipulate, annotate what's happening. Wheat does #1 but not #2 or #3.

Learning Objectives

What Should a Viewer Learn?

1

UNDERSTAND that a Wheat sprint converts open-ended questions into typed, evidence-graded claims

2

IDENTIFY the six claim types (constraint, factual, estimate, risk, recommendation, feedback) and when each is used

3

EXPLAIN how evidence tiers work (stated → web → documented → tested → production) and why upgrading matters

4

RECOGNIZE the conflict-resolution pattern: when claims contradict, challenges force resolution

5

DESCRIBE how the compiler enforces quality gates before producing output artifacts

Follows Bloom's taxonomy (lower → higher order). Measurable via self-check quiz at replay end.

Recommendations

Four Design Changes

r065 — Contextual Annotations
Tooltips on key frames
First claim: "This is a [type] claim." Evidence upgrade: "Upgraded because [artifact] verified it." Conflict: "These claims contradict — /challenge surfaces tensions."
r066 — Three-Act Scaffolding
Explore → Complicate → Resolve
Act 1: Claims and types. Act 2: Evidence, conflicts, challenges. Act 3: Compiler gates and output. Optional 1-question check between acts.
r067 — Dual Mode
Learn mode + Review mode
Learn: annotations on, gating enabled, slower autoplay. Review: annotations off, free scrubbing, full speed. Prior art unanimously uses this pattern.
r068 — Sandbox Mode
Create your own claim
Let learners create a mock claim, set type and evidence tier, see dashboard and compiler response. Client-side only. Highest-leverage pedagogical feature.
Compiled by Wheat

Risk

The Tutorial Trap

r067
Pedagogical scaffolding can make a tool feel like a tutorial
Developers who already understand Wheat will find gating and tooltips patronizing. Without dual-mode, pedagogical features annoy experts or are invisible to novices. LearnGitBranching, Redux DevTools, and Observable all solve this with sandbox/guided separation.

Mitigation: Default to Review mode for returning users. First-time visitors get a single prompt: "New to Wheat? Try Learn mode." One click, no forced onboarding.
