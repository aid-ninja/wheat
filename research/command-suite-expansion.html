Quality Assurance

Four feedback loops

/challenge — Devil's Advocacy
Adversarial research against a claim
Finds the strongest counter-evidence. Creates x### claims. If counter-evidence has higher evidence tier, compiler auto-resolves. Self-correction built into workflow.
/witness — External Corroboration
URL-based claim verification
Creates w### claims with relationship: full_support, partial_support, partial_contradiction, full_contradiction. Compiler tracks corroboration count.
/blind-spot — Gap Analysis
Analyzes what's NOT being claimed
Read-only scan for dependency gaps, type monoculture, echo chambers, and evidence ceilings. Uses compiler's new source_diversity and type_diversity data.
/calibrate — Prediction Scoring
Score predictions against production
Creates cal### claims at production tier. The meta-insight: validates whether evidence tiers predict accuracy. Tests Wheat itself.
Operations

Multi-sprint, multi-person

/replay
Time-travel through sprint evolution
Recompiles every git version of claims.json with the current compiler. Uses --input/--output and --diff modes. Identifies pivotal moments: phase transitions, peak conflicts, evidence jumps.

Generates an interactive timeline HTML.
/handoff
Package sprint for transfer
Reconstructs per-topic reasoning chains: constraint → research → prototype → conflict → feedback → current state. Optimized for a successor, not a decision-maker.

Distinct from /brief (decisions) and /present (persuasion).
/merge
Combine claim sets across sprints
Cross-sprint conflict detection, ID prefixing (auth-r001), topic alignment with user confirmation, evidence tier arbitrage (higher evidence wins).

Enables parallel research by independent teams.
