Security: Remote Farmer Attack Surface

Research | 2026-03-12 | 11 claims | r027-r037

Remote Farmer exposes Claude Code's permission system to the internet via Cloudflare Tunnel. This analysis maps the attack surface: what can go wrong, how likely is it, and what to do about it.

Key finding: the authentication model (128-bit token in URL) is cryptographically strong but operationally weak. The token leaks through multiple browser and infrastructure channels r029.

Infrastructure: Quick Tunnels

HIGH SSE Broken Through Quick Tunnels

Cloudflare quick tunnels do not support Server-Sent Events. The Farmer's /events real-time stream fails through the tunnel. Dashboard falls back to 2s polling only r027.

MEDIUM Quick Tunnels Are Dev-Only

Quick tunnels have a 200 concurrent request limit, no SLA, and Cloudflare explicitly marks them for "testing and development only." The random subdomain changes on every restart r028.

Authentication: Token Leakage

HIGH Token-in-URL Leaks

The auth token appears in ?token= query parameter, leaking through:

  • Browser history and address bar
  • Referer headers on any outbound navigation
  • Proxy and CDN access logs (including Cloudflare's)
  • Page source (embedded via html.replace('{{TOKEN}}', token)) r037

r029

MEDIUM No Token Rotation

Token is static for the server's entire lifetime. No expiry, no rotation. Only fix for a leak is full server restart, losing all in-memory state r034.

Positive: Brute Force Infeasible

128-bit entropy with timingSafeEqual makes brute force impossible (2^128 search space). However, no rate limiting exists on auth failures r033.

API: Abuse Vectors

HIGH Full Compromise on Token Leak

If URL+token leak, an attacker can:

  • Approve arbitrary tool executions (including rm -rf, sudo)
  • Set trust level to autonomous
  • Inject responses to /api/ask questions
  • Read all session activity data

r031

HIGH CORS Wildcard

Access-Control-Allow-Origin: * on all endpoints means any website can make authenticated API calls if it obtains the token. Combined with Referer leakage, a malicious page the user visits could call /api/decide r030.

MEDIUM /api/ask Injection

/api/decide is internet-accessible with token. An attacker who can observe the SSE stream can see pending request IDs and respond to /api/ask questions before the real user, injecting arbitrary instructions into the Claude Code session r032.

Recommended Mitigations

Phase 1: Quick Wins

  • Add Referrer-Policy: no-referrer header
  • Restrict CORS to tunnel origin only (not *)
  • Migrate token from URL query to HttpOnly, Secure, SameSite=Strict cookie
  • Add CSRF tokens for state-changing endpoints

r035

Phase 2: Infrastructure

  • Replace quick tunnel with named Cloudflare tunnel (supports SSE, Cloudflare Access for OAuth, stable subdomain)
  • Or: replace SSE with WebSocket (quick tunnels support WS)
  • Add token rotation on a timer or on-demand

r036
