Problem Statement

Remote Permission Dashboard for Claude Code

A web UI that lets you approve/deny tool calls, monitor activity, and view sprint state — from anywhere, securely tunneled.

Wheat Sprint · 2026-03-11
The Problem

Claude Code permissions are local-only

Today, Claude Code's permission prompts require you to be at the terminal. If you step away, work stops. There's no way to monitor what Claude is doing from your phone or another machine, no push notifications for permission requests, and no visual dashboard for sprint state.

For longer-running sessions — research sprints, prototyping, batch operations — this is a significant bottleneck. You're tethered to the terminal.

What We Want

A remote-capable permission layer

Approve / Deny
Permission notifications pushed to browser
When Claude Code needs approval, the dashboard shows a notification with full context. Approve or deny from any device. [d004]
Live Activity
Real-time feed of Claude's actions
See what tools are being called, what files are being read/written, and the conversation flow — all streaming live. [d004]
Sprint Views
Claim tree + dashboard toggle
Switch between permission queue, activity feed, claim dependency tree, and sprint status dashboard. [d004]
Constraints

Hard requirements

  • No Claude Code patches — must use the existing hook/permission system as-is [d001]
  • Node.js server, browser UI — minimal deps, no native app required [d002]
  • Secure tunneling — ngrok, Cloudflare Tunnel, or Tailscale with auth for remote access [d003]
  • Prototype + architecture doc — sprint delivers both a working proof-of-concept and a production recommendation [d005]
Key Questions

What we need to figure out

Integration
How do Claude Code hooks expose permission requests?
Can we intercept permission prompts via hooks and relay them to a web server? What's the IPC mechanism?
Security
Is ngrok secure enough for short sessions?
Threat model for tunneled permission approval. What auth layers are needed? How does Cloudflare Tunnel compare?
Architecture
WebSocket vs SSE vs polling?
What's the right real-time transport for permission notifications? Latency requirements? Fallback strategies?
