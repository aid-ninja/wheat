Critical Gaps

What's Missing

No Landmarks
Zero Semantic HTML
All structure uses <div>. No <header>, <main>, <nav>, no ARIA roles. Screen readers see a flat soup of text.
[r018]
No Keyboard
Mouse-Only Interaction
Claim rows use onclick on <div>. No tabindex, no role="button", no Enter/Space handlers. Keyboard users can't expand claims.
[r019]
No Live Region
Silent DOM Updates
SSE updates change the page silently. No aria-live regions. Screen readers don't know when new claims arrive or status changes.
[r020]
Remediation

Five Fixes to AA

  1. Raise --text-dim contrast
    #505a6e → #7a839a (4.5:1 ratio). One CSS variable.
    1 line
  2. Add semantic landmarks
    <header> for topbar, <main> for layout, role="status" on scoreboard.
    3 tag changes
  3. Keyboard support on claims
    tabindex="0", role="button", aria-expanded, keydown handler for Enter/Space.
    ~15 lines JS
  1. aria-live on dynamic regions
    aria-live="polite" on scoreboard and timeline. Screen readers announce updates.
    2 attributes
  2. Focus-visible outlines
    :focus-visible styles on all interactive elements. Blue outline on keyboard focus.
    ~5 lines CSS
Total Effort
~30 lines of changes
No visual impact on the existing design. All fixes are additive. [r021]
